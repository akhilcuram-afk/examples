How it works

  • Ingestion (Left): The system continuously crawls documents, chunks them into text, and tags them with security ACLs before storing them in the Vector DB.
  • Query (Right): When a user asks a question, the Orchestrator checks Entra ID for their permissions.
  • Security (Center): The DB only returns chunks the user is allowed to see.
  • Generation: The LLM creates an answer using only those safe chunks.
